# Repository Quality Checklist

Use this checklist to keep cybersecurity study repositories clean, consistent, and easy to use.

## README

- Clear title with the certification or topic name.
- Short explanation of what the notes cover.
- Table of contents or study index.
- Clear usage instructions.
- Official links where relevant.
- Disclaimer that notes are not official vendor material.

## Structure

- One readable file per module, domain, or section.
- Avoid unnecessary nested folders.
- Use consistent file naming.
- Keep public-facing files only.
- Remove temporary drafts, duplicate notes, and unused review files.

## Study Notes

- Explain concepts in beginner-friendly language.
- Include important terms.
- Include practice tasks or review prompts.
- Add quick-check questions.
- Add final review checklist.
- Keep formatting easy to scan.

## GitHub Discoverability

- Use clear repository names.
- Use accurate repository descriptions.
- Add relevant GitHub topics.
- Keep the README useful for learners.
- Update notes when exam outlines or course structures change.

## Maintenance

Review each repository monthly or when certification content changes. Check links, clean outdated wording, and keep the structure simple.
